# Agent Behavior For This Repo

These rules apply to Codex sessions working in this project.

1. Always show progress while working.
2. Before any substantial action, state the next step in 1 short sentence.
3. During multi-step work, post short status updates regularly (what is being checked, changed, or verified).
4. Never wait silently for user input.
5. If execution is blocked by approval, Plan mode, or missing confirmation, print a clear blocking message with exactly what is needed from the user.
6. When asking for input, include one concrete example of an acceptable reply.
7. After completing changes, report:
- what changed,
- what was validated,
- what still needs user approval (if anything).
